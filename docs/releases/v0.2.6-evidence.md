# v0.2.6 Release Evidence

Security acceptance: model API keys are injected through environment or deployment secrets only. The upgrade path removes legacy SQLite key overrides without reading or logging their values, and the admin contract exposes configured status instead of key material. Non-secret model settings are environment-owned; admin saves use an atomic `.env` replacement and update the current process only after the file write succeeds.

## Scenario

1. Sign in as an administrator and upload `refund-policy.md` from **Document Knowledge**.
2. Confirm the document reaches `ready`, inspect the parsed chunk, and keep it enabled.
3. Ask the customer chat for the unique refund policy sentence with no model keys configured.
4. Confirm the answer names `refund-policy.md` and returns its original three-business-day excerpt.
5. Rate the answer negatively and confirm Knowledge Review preserves the document/chunk retrieval snapshot.
6. Delete the document and confirm its file, database row, chunks, and in-memory index entry are removed.

## Known Limits

- Parsing and embedding run synchronously in the Express process behind administrator authorization and fixed resource caps; worker isolation is deferred.
- PDF requires a text layer. OCR, image knowledge, formal citations/page jumps, reranking, no-evidence refusal, web ingestion, and document versioning remain out of scope; v0.2.6 exposes compact document/chunk/page metadata only.
- SQLite plus the in-memory vector index remains the intended small-deployment path; no external vector database is required for v0.2.6.
