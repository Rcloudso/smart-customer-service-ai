# v0.2.5 — Knowledge Gap Feedback Loop

v0.2.5 turns weak customer-service answers into reviewable knowledge. A completed chat with no useful FAQ match, a top retrieval score below `0.55`, or a 1–2 star rating can now create one deduplicated knowledge-review item for the exact user turn.

## Product Impact

- Admins can open **Knowledge Review** to filter pending, converted, and dismissed items.
- Each item preserves the user question, assistant answer, intent, rating, and up to three retrieval results from answer time.
- Admins can edit an item into an FAQ. The conversion links the review to the FAQ, synchronizes the index, and can be retried without creating duplicate FAQs.
- Admins can dismiss items with an optional reason.
- Explicit requests for human support stay in the escalation workflow instead of becoming automatic knowledge gaps.

## Compatibility And Data

- Existing chat SSE event types and required fields are unchanged.
- Satisfaction requests may send `messageId` alone or pair it with `sessionId` for ownership validation; legacy session-only requests still rate the latest assistant message.
- Explicit human-support requests create an escalation before answer generation, so an LLM streaming failure does not drop the handoff request or create a knowledge-review item.
- The SQLite change is additive. `knowledge_review_items` and its indexes are created with `IF NOT EXISTS`; existing tables are not rewritten.
- FAQ remains the only knowledge source in this release. The stored retrieval snapshot uses generic knowledge identifiers so future document, web, OCR, and image sources can reuse the review workflow.

## Known Limits

- The review threshold is fixed at `0.55` and is not configurable in the admin UI.
- FAQ conversion first creates an inactive database record. If index preparation fails, the review remains pending and retry reuses the linked FAQ; dismissing the failed conversion removes the inactive FAQ.
- SQLite plus the in-memory `VectorStore` remains the default small-deployment path.
- Anonymous chat ownership and local-storage admin tokens remain MVP constraints.
- v0.2.5 does not add document upload, OCR, image RAG, live-agent messaging, multi-agent orchestration, MCP, or external vector storage.
