# v0.2.7 Release Evidence

This file records verification for the release branch. It does not claim a
GitHub tag or Release until those artifacts exist.

## Scenario

1. Ask a question that exactly matches an enabled FAQ.
2. Confirm the deterministic answer displays `Direct FAQ` and
   `Evidence threshold met` together with the FAQ reference.
3. Reload the conversation and confirm the same decision and source are
   restored from persisted message evidence.
4. Ask a recognized unsupported cancellation/refund action and confirm the
   server refuses without calling the answer-generation stream and creates a
   human escalation.
5. Ask a question with no supporting knowledge and confirm a deterministic
   no-evidence refusal without answer generation.

## Adversarial Review

Three independent adversarial passes checked the release contract, repository
standards, public evidence, security, compatibility, data consistency, failure
behavior, and fresh-clone usability. Their counterexamples were added as
regressions, including:

- policy/how-to questions versus private state or action requests in Chinese
  and English, including compound and negated wording;
- alias queries over conflicting direct FAQs;
- numeric normalization for decimals, ranges, signs, percentages, and
  thousands separators;
- empty or failed generation streams and restored source snapshots.

After the fixes and focused reruns, all three reviewers reported no blocking
findings. The final standards and specification reviews also covered
idempotency ownership isolation, response callback overloads, fail-closed
disconnect handling, capture memory bounds, and rating retry behavior.

## Known Limits

- Initial grounding thresholds still need the v0.2.8 quality-lab calibration.
- Arbitrary contradictions across document prose are not inferred.
- Escalation persistence is implemented; real-time human assignment is not.
- The default deployment remains a single API instance with SQLite and an
  in-memory vector index.
- Idempotency records are retained for 24 hours in the deployment SQLite
  database. Multipart FAQ import uses a synchronous UI submission lock rather
  than generic server response replay; document upload also has SHA-256 content
  duplicate protection.
